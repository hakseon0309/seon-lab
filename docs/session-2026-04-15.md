## 변경 파일

### 제거
- `src/app/signup/page.tsx` — `/login`으로만 redirect하던 stub. 어떤 버튼도 도달하지 못함.
- 미들웨어의 `/signup` public route 항목 제거.

### 신규
- `src/lib/i18n.tsx` — 모든 사용자 노출 문자열을 한 파일에서 관리하는 i18n 컨텍스트. `useI18n()` 으로 `t.*` 사용. `localStorage`에 locale 저장.
- `src/lib/supabase/admin.ts` — service-role 클라이언트 + `requireAdmin()` 가드.
- `src/app/api/admin/users/route.ts` — 사용자 목록 + 팀 매핑.
- `src/app/api/admin/team-members/route.ts` — POST/DELETE 로 팀원 추가/제거.
- `src/app/admin/page.tsx` — 관리자 UI.
- `supabase/session-2026-04-15.sql` — 본 세션 DB 변경 통합본 (지점명 수정 + `is_admin` 컬럼 + 관리자 부여).

## 운영자 액션 (반드시 수행)

1. **Supabase SQL 실행**
   - Supabase Dashboard → SQL Editor → `supabase/session-2026-04-15.sql` 전체 붙여넣고 Run.
   - 본인 계정 이메일이 `hakseon0309@gmail.com` 가 아니면 SQL 마지막 블록의 이메일을 수정.

2. **환경 변수 설정**
   - `SUPABASE_SERVICE_ROLE_KEY` — 관리자 API용 (이미 존재하면 그대로).
   - `NEXT_PUBLIC_CORP_TEAM_CODE` — 회사 팀의 `invite_code`. 미설정 시 회사 팀 버튼은 자동으로 숨겨집니다.

3. **i18n 사용**
   - 추가 언어가 필요하면 `src/lib/i18n.tsx`의 `dictionaries` 객체에만 항목을 추가/수정하면 됩니다.
   - 점진적 마이그레이션을 위해 인프라만 도입했고, 기존 화면 텍스트는 한국어 리터럴 그대로 유지. 스위치는 `useI18n().setLocale("en")` 호출로 가능.
